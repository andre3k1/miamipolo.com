Installation

In order to install this theme you must have a working version of WordPress already installed

You can install this theme in 2 ways: via FTP or via WordPress theme upload function.

Way 1: FTP installation

  • Connect to your hosting via an FTP software
  • Unzip the Grata.zip file on your computer
  • Upload the extracted GRATA theme folder into wp-content > themes in your wordpress installation

Way 2: WordPress installation

  • Navigate to Appearance > Themes.
  • Click Install Themes and then press the upload button.
  • Find the Grata.zip file on your computer and click Install Now button.

After the installation GRATA Theme must appear at Appearance > Themes page, where you need to activate it.

After you activate the theme, it is highly recommended to import the chosen demo data to speed up the website development process. Please read the following section for more info.

Demo Data Import

You can quickly make your site look same way as one of our demos.

To do so, go to Appearance > Demo Import page, select wanted demo and press the Import button. Wait until import process is finished and that's it.

 

 

Optional: in case something went wrong with automatic import, follow the steps below:

  • The XML files are included to theme files. They are located in a folder called xml
  • Navigate to WP Admin > Tools > Import
  • Select and install WordPress importer
  • Once its installed, select corresponding XML file and upload it. It will take a few minutes to install and upload all content
  • Navigate to WP Admin > Appearance > Menus page and set menus to apropriate locations
  • Go to Settings >Reading page and set Front Page to display a Static page, then select page named 'Grata Home' to be the Front Page
XML file doesn't include images, sliders and advanced theme options settings. See corresponding sections for more information.

Main Page Arrangement

GRATA Theme Main Page is built with "Page Sections" posts and Main Menu. You can see how it is done if you import one of our demos. If you want to build your site from scratch, following information may be useful for you:

In order to build your Main Page first create it's sections: use Page Sections > Add Page Section admin page and create as many sections as you need to represent your service or product.

Then proceed to Appearance > Menus. Create a new menu and set in Theme Locations box as Main Menu. Then choose Page Sections that you want to see at your Home Page and add them to Menu (if you do not see the Page Sections list under Pages list, open the Screen Options in top right corner of Admin Panel and check them to appear).

Order of Page Sections in Top Menu corresponds to the order they appear on your Main Page. So you can arrange them as you want.

If you want to add more then one Page Section to represent something - you can use parent-child relations for these Page Sections. For example: first Page Section with Contact information with parallax background, second - with Contact form and  third - Full Width Page Section with Google Map will be represented as 'Contact Info' on your Menu.

Create first Page Section, fill it with info, then add second Page Section and set first one as Parent for it - see Attributes meta box, then do same for thight Page section. Then proceed to Appearance > Menus and add first Page Section to Main Menu and save it. You will see all three Page Sections were added to your Main Page.

Portfolio Arrangement

GRATA Theme provides possibility to present your portfolio in a simple way.

First, create your Portfolio Items by using Portfolio > Add Portfolio Item page in Admin area.

Then you can arrange your Portfolio Items by adjusting their Publish Date. You can adjust it at Publish area at Edit page of Portfolio Item. The newer the date, the higher Portfolio Item will be placed in list.

After creating and arranging your Portfolio Items, just create a new Page Section or edit already created one and place [portfolio] shortcode to it's content.

[cols]

Container for one row of responsive columns.

Attributes and possible values:

  • offset:
    • default (or leave blank) - default behavior
    • wide - make indents between columns wider

[one_half], [one_third], [two_third], [one_quarter], [three_quarter], [one_fifth], [two_fifths], [three_fifth], [four_fifth], [one_sixth], [five_sixth] - responsive columns.

Example:

[one_half] ... [/one_half]
[one_quarter] ... [/one_quarter]
[one_quarter] ... [/one_quarter]
[/cols]

[portfolio]

Displays grid of your Portfolio Items.

Attributes and possible values:

  • columns - 2, 3, 4, 5
  • items - quantity of portfolio items to fetch. If left blank, equals amount of Columns
  • style - Item view style: type_1, type_2, type_3, type_4, type_5, type_6
  • align - alignment of Item title and meta text: left, center, right
  • ratio - 3:2, 4:3, 1:1, 2:3, 3:4
  • indents - 1 or 0 (yes or no respectively) - whether to show indents between items
  • meta - date, category or leave blank to display no meta text below title
  • filters - 1 or 0 (yes or no respectively) - whether to use filters by category
  • category - you can set category to fetch items from it only
  • pagination - 1 or 0 (yes or no respectively) - whether to use pagination
  • items_bg_color (optional) - custom color for items' background
  • items_text_color (optional) - custom color for items' text

Example:

[portfolio columns="5" items="" style="type_1" align="center" ratio="3:2" indents="0"
 meta="" filters="0" category="" pagination="0" items_bg_color="" items_text_color=""]

[blog]

Displays blog posts.

Attributes and possible values:

  • type - type of blog list view:
    • square - list with square thumbs at the left
    • rounded - list with rounded thumbs and date in circle at the left
    • masonry - masonry grid
  • columns - 1, 2, 3
  • items - number of posts to fetch
  • show_date - 1 or 0 (yes or no respectively)
  • show_author - 1 or 0 (yes or no respectively)
  • show_categories - 1 or 0 (yes or no respectively)
  • show_tags - 1 or 0 (yes or no respectively)
  • show_comments - 1 or 0 (yes or no respectively)
  • show_read_more - 1 or 0 (yes or no respectively)
  • category - you can set category to fetch posts from it only
  • pagination - 1 or 0 (yes or no respectively) - whether to use pagination

Example:

[blog type="rounded" columns="2" items="4" show_date="1" show_author="1" show_categories="0" 
show_tags="0" show_comments="1" show_read_more="0" category="" pagination="0"]

[gmaps]

Attributes and possible values:

  • address - simply entering an address will position map at needed point of the map
  • longitude and latitude - if you cannot position the map correctly by using just address, this attributes come in handy - you can presisely set the position.
  • marker (Marker text) - when entered, baloon with text from this attribute will appear at the center of map
  • height - you can alter the height of map using this attribute. Width of map depends on where it's placed - so if you want to make it take 1/2 of page, place the map into 1/2 column.
  • type (Mat type) - four types is available:
    • ROADMAP - Roadmap
    • SATELLITE - Satellite
    • HYBRID - Map + Terrain
    • TERRAIN - Terrain
  • zoom - a number from 1 to 20. 14 by default

Example:

[gmaps address="1600 Amphitheatre Parkway, Mountain View, CA 94043, United States" 
latitude="" longitude="" marker="" height="400" type="ROADMAP" zoom="14"]

Translating GRATA

You can translate GRATA into your language using files located at Grata/languages folder. In order to translate the theme, follow the steps below:

  • Locate default.po and default.mo files at Grata/languages folder
  • Edit the .po file using POEdit
  • Use the translation field to make replacements
  • Save file with your language name (File > Save as at program menu), for example: es_ES.po. It will generate both a .po and .mo file for your translation.
  • Go to your server via FTP, and upload the language files to the languages folder of Grata theme (located at wp-content/themes/Grata/languages)
  • Edit wp-config.php located in the root folder of wordpress and define the WPLANG constant with your language name which should be similar to the translation file name (in this example 'es_ES')

GRATA is WPML ready

If you need a Multi-Language site, then we recommend using the WPML plugin. You would need to buy a license for it, then install it and set it up. All information and documentation can be found at plugin's website: WPML.org
